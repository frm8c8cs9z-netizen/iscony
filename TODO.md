- 当日受付・結果入力用に、リーグ試合とトーナメント試合を同じ画面で探せる入口を作る。
  - 主な探し方は、スコアシートに印字された `カテゴリ + 番号` と `コート + 第何試合` を想定する。
  - 第1段階として、`カテゴリ + 番号` と `コート + 第何試合` でリーグ/トーナメントを横断検索し、結果入力へ進める画面を追加済み。
  - 使ってみた感触として、フォーム検索は1回戦目など受付が押しかける時間帯の主導線としては重い可能性が高い。
  - 現行検索画面は、迷った時・例外対応・進行表と紙が合わない時の補助導線として据え置く。
  - 混雑時の主導線は、進行表から直接押す導線、短い検索キー、QR読み取りを組み合わせて育てる。
  - 補助的に、選手名、ペアコード、Stage、リーグ名、トーナメント名、試合番号でも検索できるようにする。
  - リーグとトーナメントを同じ一覧に表示し、結果未入力・進行中・完了で絞り込めるようにする。

## Stage反映・スナップショット運用

- スナップショットから「結果のみ復元」できるようにする。
  - 目的は、進行表・枠配置・AdvancementSourceなどは現在のまま維持し、誤入力した試合結果や順位だけを戻すこと。
  - 復元対象の候補は、リーグ試合結果、トーナメント試合結果、リーグ順位、リタイア結果、追加試合結果。
  - 進行状態（呼出、試合中、完了）を結果のみ復元に含めるかは要検討。現場では「進行は今のまま、点数だけ戻す」需要がありそう。
  - トーナメント結果を戻す場合、勝ち上がり先の枠や後続Stage反映済み枠との整合性チェックが必要。
  - UIでは復元範囲を選びすぎると難しくなるため、まずは「カテゴリ単位」「結果のみ」のシンプルな入口から検討する。
- KENKOCUP_2026 女子Bの予選リーグ結果入力後、以降Stageの情報を途中まで再読み込みした際の挙動を確認する。
  - 現象: 予選リーグの枠だけが残り、結果や選手の貼り付け位置が消えた。
  - この挙動が仕様として妥当か、再読み込み時の削除範囲・保護範囲を確認する。
  - 実運用では結果入力済みStageや反映済み枠が失われないよう、バリデーションまたは確認導線が必要か検討する。
- `settings.py` の `ENABLE_AUTO_OPERATION_SNAPSHOT = True` でも自動スナップショットが作成されていない原因を確認する。
  - 想定トリガは後続Stage反映前。
  - 設定値、呼び出し経路、同一内容スナップショット抑制、エラーメッセージの有無を確認する。
- 大会ごとに運用パラメータを持てるようにする。
  - 大会ごとにオペレーションが異なるため、自動Stage反映、自動スナップショットなどを大会単位で設定可能にする。
  - 将来的には `Tournament` または大会設定モデルに運用フラグを持たせる。
- 自動Stage反映を実装する。
  - まずは手動反映と同じ制約を満たす場合だけ実行する。

  - 自動スナップショットとの連動も考慮する。
