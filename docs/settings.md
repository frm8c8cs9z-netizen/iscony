# Settings

このメモは、`stpro/config/settings.py` に追加している運用パラメータの意味と調整目安をまとめる。

## `PUBLIC_VIEW_CACHE_SECONDS`

一般参加者向けの読み取り専用画面だけに使う短時間キャッシュ秒数。

対象画面:

- `public_category_results`: 公開カテゴリ結果画面
- `public_schedule_view`: 一般参加者向け進行表

対象外:

- 管理メニュー
- 結果入力画面
- 試合進行の管理者向け画面
- CSV取込、スナップショット復元などの更新系画面

設定例:

- `0`: キャッシュなし。結果反映を最優先する場合。
- `5`: 標準値。結果反映の遅れを抑えつつ、参加者アクセスの集中を軽減する。
- `10`: 大規模大会やアクセス集中時の負荷軽減を優先する場合。
- `15` 以上: サーバ負荷がかなり高い場合の緊急寄り設定。結果反映の遅れを参加者が感じやすくなるため注意。

考え方:

- 公開画面は参加者・関係者が同じURLを何度も見るため、短時間キャッシュの効果が大きい。
- 当日運営側の画面は、結果入力や進行変更の即時性が重要なのでキャッシュしない。
- まずは `5` 秒を標準とし、負荷が高ければ `10` 秒へ上げる。

## `ENABLE_AUTO_OPERATION_SNAPSHOT`

後続Stageへ結果反映する直前に、大会全体スナップショットを自動作成するかどうか。

設定例:

- `True`: 自動作成する。通常はこちら。
- `False`: 自動作成しない。スナップショットが増えすぎる場合や、手動運用に寄せたい場合。

考え方:

- 後続Stage反映は大会構造に影響するため、事故対応のために自動スナップショットを残す。
- 容量や件数が問題になる場合は、運用整理とあわせて無効化を検討する。
