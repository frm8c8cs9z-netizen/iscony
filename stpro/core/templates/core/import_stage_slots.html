CSV仕様

カンマ区切りCSVとタブ区切りCSVの両方を読み込めます。 stage_type は L または T、source_type は LR または TR を指定します。

CSVに含まれるStageだけを置き換えます。初回はCSVでの登場順をStageの表示順とし、 再取込では既存Stageの順番を維持します。新しいStageはカテゴリ内の末尾へ追加します。 GroupとBracketは各Stage内での登場順になります。

stage_codeは任意です。空欄の場合はアプリがSTG1、STG2のように自動発番します。 既存のstage_codeを指定して異なるstage名を読み込むと、同じStageの名称を変更できます。 source_stage_codeを指定すると、進出元Stageも名称に依存せず参照できます。

列名 内容
category / stage_code / stage / stage_type カテゴリ、任意のStageコード、ステージ名、L/Tです。
group / bracket Lではgroup、Tではbracketを指定します。
slot_code / display_order 枠番号と表示順です。
entry_code 参加者を直接入れる枠で使います。
source_type LRはリーグ順位、TRはトーナメント結果です。
source_stage / source_stage_code / source_group / source_rank LRで使用します。source_stage_codeを指定した場合は、source_stageより優先します。
source_stage / source_stage_code / source_bracket / source_match / source_result TRで使用します。source_resultは win または lose です。同じステージ内にM1が複数ある場合はsource_bracketが必要です。
