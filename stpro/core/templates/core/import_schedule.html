CSV仕様

リーグとトーナメントの進行順を同じCSVで登録できます。 旧リーグ進行CSVも、対象リーグを一意に特定できる場合は利用できます。

列名 内容
schedule_block 任意。開催日や時間帯を分ける名前です。例: 1日目、2日目。空欄時は「本日程」を使用します。
category 必須。大会に登録されているカテゴリ名です。
stage_code 推奨。対象Stageの固定コードです。Stage名を変更しても同じStageを特定できます。
stage 任意。stage_codeを使わない場合に対象Stage名を指定します。
stage_type 旧形式CSVとの互換用です。新しいCSVフォーマットには含まれず、stage_codeから自動判定します。
group / pair1 / pair2 旧形式CSVとの互換用です。新しいCSVフォーマットには含まれませんが、従来どおり取り込めます。
bracket トーナメント行で使用するトーナメント表名です。同じステージ内に複数の表がある場合は必須です。
court 必須。試合を割り当てるコート名です。未登録のコートは自動作成します。
order 必須。日程区分内における、そのコートの試合順です。1以上の整数を指定します。
slot1_code リーグ行では必須。対戦する一方のslot_codeです。
slot2_code リーグ行では必須。対戦するもう一方のslot_codeです。2枠は同じGroupに所属している必要があります。
meeting_number 任意。同じ2枠による何回目の対戦かを指定します。空欄時は1です。追加対戦は2以上を指定します。
match_code トーナメント行では必須。対象トーナメント表内の試合コードです。例: M1。
match_label 任意。トーナメント試合の表示名です。例: ①。内部のmatch_codeは変更しません。
match_games 任意。ゲーム数です。リーグ行で空欄の場合は7、トーナメント行で空欄の場合は現在の設定を使用します。
